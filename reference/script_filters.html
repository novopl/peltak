Built-in script template filters¶

Built-in jinja2 filters¶

You can find the list of all filters built into jinja2 in jinja2 documentation

Filters provided by peltak¶

peltak.extra.scripts.filters.header(title)[source]¶

Converts a given title into a pretty header with colors.

Usage:

{{ 'hello' | header }}

will result in:

= hello ================================================================

The resulting string will be colored for printing in the terminal.

peltak.extra.scripts.filters.count_flag(count, flag)[source]¶

Returns the given flag letter as a count flag.

Usage:

{{ set verbose = 3 }}
{{ verbose | count_flag('v') }}

will result in:

-vvv

In the above example, if verbose is 0 the result of this filter will be an empty string.

peltak.extra.scripts.filters.cprint(msg, *args, **kw)[source]¶

Will convert the given message to an echo statement with color opcodes.

This supports the same syntax as shell.fmt (used internally here). The color processing will replace any tag like object in format <NUMBER> into a corresponding opcode. Here’s a quick cheatsheet on some of the more usefull opcodes

Tag Description
<0> Reset all formatting to default values.
<1> Intensify current color (will affect all other color opcodes).
<31> Set text color to red.
<32> Set text color to green.
<33> Set text color to yellow.
<34> Set text color to blue.
<35> Set text color to pink.
<35> Set text color to teal.

Usage:

{{ '<35>hello, <32>world' | cprint }}

will result in the following:

echo "\x1b[35mhello, \x1b[32mworld\x1b[0m"

Which inside a terminal will be rendered as hello in ping and world in green.

cprint also supports formatting, same as in the built-in format() function.

{{ "hello, {}, I'm {name}" | cprint('Susan', name='John') }}

will result in:

echo "hello, Susan, I'm John\x1b[0m"
